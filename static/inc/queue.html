Queue

Parameter Description
attrs Comma-separated list of attribute names to include in the response. If omitted, all attributes are returned.
count If set to true, the response will include the X-Total-Count HTTP header indicating the total number of matching entities.
coordinates Used with geo-queries. Specifies a geometry area (like a polygon, point, or bounding box) to apply the spatial filter.
csf Context Source Filter: used when federated context sources are in play. Filters which sources to query.
format Output format. Possible values: normalized, keyValues, or geojson. Affects the structure of returned attributes.
geoproperty Specifies the name of the GeoProperty to use for spatial filtering (default is "location").
geometry Type of geometry used for spatial filtering: Point, LineString, Polygon, MultiPolygon, BBox, etc.
geometryProperty Synonym for geoproperty; used to declare the geo-attribute used for georel.
georel Defines the spatial relationship to be applied in the geo-query. Examples: near;maxDistance==1000, within, intersects.
id One or more entity IDs (comma-separated) to filter results.
idPattern A regex pattern to match entity IDs. Useful for retrieving a group of entities with a naming convention.
lang Specifies the language for multilingual properties (if LanguageProperty is used).
local If true, restricts the query to only local entities (ignores federated sources or linked data contexts).
limit Max number of entities to return. Useful for pagination.
offset Index of the first entity to return. Used in conjunction with limit for pagination.
onlyIds If true, returns only the entity IDs instead of full entities.
options A list of flags to modify query behavior. Common values: keyValues (flattened response), sysAttrs (include system attributes).
q Advanced query expression for attribute values (e.g., temperature>25;status=="active").
scopeQ Used for scoping in linked data and federated querying. Often not needed in single-broker deployments.
type One or more entity types (comma-separated) to filter the results.
pick Used to select only certain properties (e.g., pick=temperature,humidity), similar to attrs, depending on broker version.
